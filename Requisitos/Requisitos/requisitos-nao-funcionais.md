# Requisitos Não Funcionais

## 1. Introdução

Este documento apresenta os requisitos não funcionais da plataforma GAC, relacionados a desempenho, segurança, disponibilidade, usabilidade e integração do sistema.

---

## 2. Requisitos Não Funcionais

### RNF01 - Desempenho

O sistema deve responder operações de consulta em até 3 segundos.

---

### RNF02 - Disponibilidade

O sistema deve estar disponível durante o horário de funcionamento da instituição.

---

### RNF03 - Segurança

O sistema deve exigir autenticação para acesso às funcionalidades administrativas.

---

### RNF04 - Controle de acesso

O sistema deve restringir funcionalidades conforme o perfil do usuário.

---

### RNF05 - Persistência de dados

Os dados dos projetores e movimentações devem ser armazenados em banco de dados persistente.

---

### RNF06 - Auditoria

O sistema deve manter logs de operações realizadas pelos usuários.

---

### RNF07 - Compatibilidade NFC/RFID

O sistema deve suportar integração com dispositivos de leitura NFC/RFID.

---

### RNF08 - Usabilidade

O sistema deve possuir interface web intuitiva e responsiva.

---

### RNF09 - Portabilidade

O sistema deve funcionar em navegadores modernos e dispositivos móveis.

---

### RNF10 - Integridade de dados

O sistema deve garantir integridade das informações registradas sobre os projetores.

---

### RNF11 - Escalabilidade

O sistema deve suportar expansão futura para outros tipos de ativos além de projetores.

---

### RNF12 - Confiabilidade

O sistema deve minimizar perda de dados em falhas de operação.
